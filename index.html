C’est-à-dire qu’il est possible de concevoir en Python des entités qui miment celles du monde réel (une cellule, une protéine, un atome, etc.) avec un certain nombre de règles de fonctionnement et d’interactions ; - Il est relativement simple à prendre en main2 ; - Enﬁn , il est très utilisé en bio-informatique et plus généralement en analyse de données. Variables

Déﬁnition

Une variable est une zone de la mémoire de l’ordinateur dans laquelle une valeur est stockée. Aux yeux du programmeur, cette variable est déﬁnie par un nom, alors que pour l’ordinateur, il s’agit en fait d’une adresse, c’est-à-dire d’une zone particulière de la mémoire. En Python, la déclaration d’une variable et son initialisation (c’est-à-dire la première valeur que l’on va stocker dedans) se font en même temps.

Chapitre II: Algorithme

I/ Pincipe clés

a/ La description algorithmique des traitements de programme

Chaque traitement doit être traduite par une suite d’instructions de programme pour pouvoir s’exécuté sur un ordinateur. Un programme est un objet très technique et détaillé, qui fait appel aux formulations d’un langage de programmation. Avant de réaliser un programme, il est nécessaire de définir avec précision ce qu’il doit faire. Un algorithme remplit cette fonction en décrivant la logique du programme sous forme standardisée et facilement accessible. Le langage structuré présente quatre avantages : - Il facilite, par sa syntaxe, l’utilisation de bonnes structures ; - Il est facile à mettre en œuvre sans outil de dessin ; - Il produit des algorithmes compacts ; - Il se rapproche de la formulation proposée par les langages de programmation évolués. Attention : la syntaxe utilisée en python pour un bloc de code dans une fonction pour mettre le bloc de code à l’intérieur de la fonction est la tabulation (c’est la touche à gauche de la lettre « a » de votre clavier) dans les autres langages c’est les accolâtes qui font foi.

b/ La structue générale d'un algorithme

Un algorithme est encadré par les mots-clés DEBUT et FIN. Il comprend classiquement quatre sections : - Les déclarations qui incluent les déclarations des noms attribués aux variables utilisées dans l’algorithme et la définition de leurs types ; - Les initialisations, c’est-à-dire l’affectation de valeurs initiales à certaines variables, ou la description d’actions préparant le traitement ; - Le traitement proprement dit ; - L’arrêt du traitement, qui peut nécessiter des actions particulières afin de laisser la base de données dans un état cohérent ou de fournir les derniers résultats. Dans certains cas particuliers, où l’algorithme illustre la logique générale d’un traitement sans prétendre à l’exhaustivité, les déclarations peuvent être omises.

b/ Les structures algorithmiques

b.1/ La structure alternative

La structure alternative indique que deux actions différentes sont menées selon q'une condition est ou non remplie. La condition peut être simple ou composée. L’expression d’une alternative en langage algorithmique en python est la suivante : - try (essai) « description d’exécution d’une condition » - except (exception) « description d’action à exécuter si condition fausse » - else (sinon) «description d’action à exécuter si condition vraie »

Veuillez exécuter ce programme en mettant kaboré pour le nom et paul pour l'age et observer le résultat : nom = input ("Quel est votre nom?") age = input ("Quel est votre age?") try: age_prochain = int(age) + 1 except: print("Erreur : vous devez rentrer un nombre pour l’âge") else: print(" Vous vous appelez " + nom + ", vous avez " + str(age) + ",ans") print(" l'an prochain vous avez " + str(age_prochain) + ",ans")

Attention : cette fois ci les deux print sont à l’intérieur du else

Vous voyez que cette fois ci, nous avons un message très claire pour l’utilisateur

c/ La structure répétive ou itérative

c.1/ Boucle While (tantque) et la boucle For (pour)

La structure répétitive indique qu’une action doit être répétée. Jusqu’au moment où les conditions sont remplies pour passer à la suite. Son expression la plus générale est, en langage algorithmique : TANTQUE « expression de condition » « Liste des actions à exécuter quand la condition est remplie » Le « TANTQUE » autorise ou arrête l’exécution des actions à répéter en fonction de la valeur de la condition. La fin « TANTQUE » marque la fin des actions à répéter et renvoie au « TANTQUE » du début, donc à un nouveau test de condition.

c.2/ L'exploitation séquentielle d'un programme

La structure répétitive « TANQUE » peut décrire l’exploitation séquentielle, ligne par ligne, de l’ensemble d’un programme. Par convention, nous interpréterons le balayage d’un programme par « TANQUE » de la manière suivante : -le DEBUT de l’algorithme « ouvre » le programme et positionne le traitement sur ma première ligne ; -une lecture ou une écriture du programme dans la boucle « TANQUE » concerne la ligne en cours ; -chaque passage sur la fin de la boucle « TANQUE » provoque le passage de la ligne suivante ; -la boucle « TANQUE » se termine quand la dernière ligne du programme a été traitée. Des actions particulières peuvent être nécessaires pour initialiser ou clore les traitements, elles consistent notamment : -à lire la première ligne ; -à mener des actions terminant les traitements après le traitement de la dernière ligne.

Exemple: reprenons notre programme nom = input ("Quel est votre nom?") age = input ("Quel est votre age?") try: age_prochain = int(age) + 1 except: print("Erreur : vous devez rentrer un nombre pour l’âge") else: print(" Vous vous appelez " + nom + ", vous avez " + str(age) + ",ans") print(" l'an prochain vous avez " + str(age_prochain) + ",ans")

Exécutez ce programme et appuyer sur entrer sans rien saisir au niveau de l'age, vous voyez qu’il vous redemande systématiquement l'age.
